# ADR-0001: Standardize on Python 3.12

- Status: Accepted
- Date: 2026-07-23
- Owners: Project maintainers
- Related issues/PRs: Milestone M0

## Context

The API, worker, deterministic ML modules, evaluation tools, and generated-code sandbox all
depend on Python. Allowing different minor versions would make dependency resolution, model
artifacts, local development, CI, and sandbox reproduction less reliable.

Python 3.12 provides a modern, stable baseline while retaining broad support across FastAPI,
Pydantic, pandas/Polars, scikit-learn, XGBoost, LightGBM, Celery, and MLflow ecosystems.

## Decision

Use Python 3.12.x for all Python code in this repository.

- Keep `.python-version` at `3.12`.
- Declare `requires-python = ">=3.12,<3.13"` in Python packaging metadata.
- Generate the dependency lockfile using Python 3.12.
- Run Python CI jobs on Python 3.12.
- Build API, worker, ML, evaluation, and sandbox images with Python 3.12.
- Pin deployable images to an exact patch release and immutable digest.

Do not add Python 3.11, 3.13, or another minor version to the supported matrix without a new
ADR and compatibility evidence.

## Alternatives considered

### Support multiple Python versions

This improves library-style compatibility but increases CI, dependency, serialization, and
debugging complexity without meaningful MVP value.

### Use the newest available Python minor automatically

This reduces deliberate maintenance but makes builds less reproducible and can expose the
project to ecosystem compatibility gaps.

## Consequences

- Local development, CI and containers share one minor-version baseline.
- Contributors must install a Python 3.12 interpreter.
- Dependency upgrades are evaluated against Python 3.12.
- Moving to another minor version is an explicit migration with lockfile, test, model-artifact,
  sandbox-image, and benchmark validation.

## Validation

- CI verifies the interpreter reports Python 3.12.
- Packaging rejects Python versions outside `>=3.12,<3.13`.
- Container smoke tests report Python 3.12.
- Reproducibility bundles record the full Python patch version and image digest.

## Revisit when

Revisit when a critical dependency drops Python 3.12, a required security policy mandates a
newer runtime, or the project reaches a planned platform-upgrade milestone.
